# PokemonStadiumRecomp

Static recompilation of **Pokémon Stadium (US v1.0)** to native PC.
Built on top of [N64Recomp](https://github.com/N64Recomp/N64Recomp).

> **Status:** boots through the boot logo, intro sequence, title screen,
> area select, and into in-game scenes. Audio synthesis runs.
> Several crash families remain under active investigation in private
> branches. Pre-built Windows binaries are attached to GitHub Releases;
> the build is not currently reproducible from this snapshot alone (the
> engine forks live in private branches that haven't been pushed yet).

## ROM (you supply your own)

This project is **bring-your-own-ROM**. No copyrighted bytes ship in
this repository or in the released binary. You must legally own a
Pokémon Stadium cartridge and produce your own `baserom.z64` dump.

| Field | Value |
|-------|-------|
| Title | Pokémon Stadium (US, v1.0) |
| MD5   | `ed1378bc12115f71209a77844965ba50` |
| Size  | 33,554,432 bytes (32 MB) |
| Format | `.z64` (big-endian native, magic `80 37 12 40`) |

**Rev A (v1.1) is not compatible** — pret's disassembly targets v1.0
specifically, and the address tables in `disasm/yamls/us/rom.yaml`
will not align with a Rev A binary. If you have Rev A, find a v1.0
dump.

Place your `baserom.z64` at the project root before building or
running. The runner verifies the MD5 at startup and refuses to launch
on mismatch. The build pipeline runs the same check.

## Quick start (running the released binary)

1. Download the latest release from the Releases page.
2. Drop your own `baserom.z64` (US v1.0, MD5 above) next to
   `PokemonStadiumRecomp.exe`.
3. Run `PokemonStadiumRecomp.exe`.

## Quick start (building from source)

Prereqs: git, python3, cmake 3.20+, a working C/C++ toolchain
(clang-cl / MSVC on Windows). For the disasm build (optional):
`make`, `binutils-mips-linux-gnu`.

```bash
# Linux / macOS / Git Bash
chmod +x setup.sh && ./setup.sh

# Windows (cmd / PowerShell)
setup.bat
```

`setup` will:
1. Clone (or junction) `n64recomp/` at the SHA pinned in
   `n64recomp.pin`.
2. Initialize the `disasm/` submodule (pret/pokestadium).
3. Stage `baserom.z64` into `disasm/baseroms/us/`.
4. Generate `aspmain_combined.bin` from your ROM (the RSP recompiler
   input — never committed, always regenerated from your own ROM).

## Layout

```
PokemonStadiumRecomp/
├── baserom.z64            # canonical ROM (gitignored, BYO)
├── aspmain_combined.bin   # RSP build input (gitignored, regenerated by setup)
├── disasm/                # pret/pokestadium submodule
├── n64recomp/             # N64Recomp engine (junctioned by setup, gitignored)
├── lib/                   # N64ModernRuntime + RT64 (gitignored, see note)
├── generated/             # recompiler C output (gitignored)
├── ghidra/                # Ghidra project slot (project state gitignored)
├── tools/                 # game-specific tooling
├── src/main/              # runner entrypoints (host integration)
├── include/               # runner headers
├── rsp/aspMain.cpp        # recompiled RSP audio ucode
├── extras.c               # recompiler patch hooks (declared in game.toml)
├── game.toml              # N64Recomp config
├── aspMain.us.1.0.toml    # RSPRecomp config
├── n64recomp.pin          # engine SHA pin
├── CMakeLists.txt
├── setup.sh / setup.bat
└── README.md
```

## Pipeline overview

```
disasm/  +  baserom.z64    -->  pret build      -->  pokestadium-us.elf
                                  (make init && make)         |
                                                              v
                          game.toml  +  N64RecompCLI  -->  generated/*.c
                                                              |
                                                              v
                                  CMake build  +  N64ModernRuntime
                                                              |
                                                              v
                                                  PokemonStadiumRecomp.exe
```

The disasm produces an ELF that already encodes every section's
load VA, symbols, and relocations. **N64Recomp consumes the ELF
directly** — there's no per-fragment slicing step.

## Out of scope (first pass)

**GB Tower / Game Boy emulation.** Pokémon Stadium contains an
embedded Game Boy emulator for Transfer Pak games (Pokemon R/B/Y).
The GB-related code lives partly in the resident `text` segment
and partly in the `gb_tower_roms` tail segment. For the first
playable build the GB Tower entry points are stubbed, not ported.

**Reproducibility from this snapshot.** This repository contains
the project sources but the engine subtree (`lib/`) is currently
not vendored or published — the released binary is built against
private working branches of N64ModernRuntime, N64Recomp, and RT64.
The plan is to publish those forks once they stabilize.
